# Vibe Coding Workflow Guide
## When to Use Each Resource, In Plain English

This guide explains **when** each resource is useful in the vibe-coding workflow, what it helps with, and how to use it with a coding agent in a practical way.

## 1. Before You Build Anything

### Use: The Mom Test
**When to use it:**  
Use this when you have an idea but you do not know if people actually need it.

**Why it matters:**  
It helps you avoid building something that sounds cool but solves no real problem.

**How to use it with a coding agent:**  
- Ask the agent to help you draft customer interview questions.
- Tell the agent not to ask leading questions.
- Ask it to turn interview answers into problem statements.
- Use the insights to decide whether to build at all.

`Use this first when you are still figuring out whether your idea is worth building.`

---

## Simple Workflow Rule

Use the resources in this order:

1. Validate the idea.
2. Write the spec.
3. Pick the stack.
4. Build the context.
5. Prompt the agent clearly.
6. Implement in small pieces.
7. Improve the UI.
8. Secure the product.
9. Launch carefully.
10. Maintain and learn.

---

## How to use a coding agent perfectly

If you want the best output from a coding agent, do this:

- Give one task at a time.
- Tell it the stack.
- Give it the spec.
- Add constraints.
- Ask for a specific output format.
- Review the result before moving on.
- Correct small mistakes immediately.
- Keep a running project memory.
- also make the Sub-agents for smaller tasks.
- Do not let it guess architecture.
- Do not let it skip testing.

The goal is not to make the AI do everything. The goal is to make the AI work inside a disciplined system.

---

## Final rule

Use the right resource at the right time.  
Do not use all resources at once.  
Use them as a workflow, not as a random list.
